OpenBoil Documentation

Complete guide to setting up, configuring, and deploying the OpenBoil SaaS boilerplate. Everything you need to launch your SaaS product in record time.

Modular Providers

Swap auth, payments, DB, and AI providers with one config change.

3 Homepage Designs

Modern SaaS, Minimal, or Bold Dark — pick one in config.

Full Auth System

Login, register, OAuth, sessions, roles — all pre-built.

AI-Powered Tools

Content generation, image creation, document analysis included.

Prerequisites

Before you begin, make sure you have the following installed:

ToolVersionPurpose
Node.js18.x or higherJavaScript runtime
npm9.x or higherPackage manager (comes with Node)
PostgreSQL14+ (optional)Required only if using Drizzle DB provider
GitAnyVersion control
💡
If you don't have PostgreSQL locally, you can use Neon (free tier) for a cloud-hosted PostgreSQL database. Or use Docker Compose which includes PostgreSQL automatically.

Installation

Clone the repository

git clone https://github.com/yourorg/openboil.git
cd openboil

Install dependencies

npm install

This installs Astro, React, Tailwind CSS, shadcn/ui, Drizzle ORM, and all other dependencies.

Run the setup wizard OR configure manually

npm run setup

The interactive wizard will ask you to choose providers and generate your config + .env file. Alternatively, see Manual Setup below.

Set up the database (if using Drizzle)

npm run db:push

This pushes the schema to your PostgreSQL database. Make sure DATABASE_URL is set in .env first.

Seed sample data (optional)

npm run seed

Creates an admin user (admin@openboil.com / password123), sample users, and sample usage data.

Start the dev server

npm run dev

Open http://localhost:4321 in your browser. You're ready to build!

Manual Setup

Step 1: Create your .env file

cp .env.example .env

Then open .env and fill in the variables for the providers you've chosen. You only need to fill in the section for your selected provider. For example, if using Custom auth + Stripe + Drizzle + OpenAI + Resend:

# Required for Custom auth
JWT_SECRET=generate-a-random-64-char-string-here

# Required for Google OAuth (optional)
GOOGLE_CLIENT_ID=your-google-client-id
GOOGLE_CLIENT_SECRET=your-google-client-secret

# Required for Drizzle + PostgreSQL
DATABASE_URL=postgresql://user:password@localhost:5432/openboil

# Required for Stripe
STRIPE_SECRET_KEY=sk_test_...
STRIPE_WEBHOOK_SECRET=whsec_...
PUBLIC_STRIPE_PUBLISHABLE_KEY=pk_test_...

# Required for OpenAI
OPENAI_API_KEY=sk-...

# Required for Resend
RESEND_API_KEY=re_...
EMAIL_FROM=noreply@yourdomain.com

Step 2: Edit openboil.config.ts

This is the central configuration file. Set your app name, choose providers, define pricing plans, and pick a homepage design:

import type { OpenBoilConfig } from "./src/config/schema";

const config: OpenBoilConfig = {
  app: {
    name: "My SaaS",
    description: "Your SaaS description here",
    url: "http://localhost:4321",     // Change to your domain in production
  },

  homepage: "modern",                 // "modern" | "minimal" | "bold-dark"

  providers: {
    auth: "custom",                   // "firebase" | "supabase" | "custom"
    payment: "stripe",                // "stripe" | "lemonsqueezy"
    database: "drizzle",              // "firestore" | "supabase" | "drizzle"
    ai: "openai",                     // "openai" | "anthropic" | "google"
    email: "resend",                  // "resend"
  },

  features: {
    blog: true,
    ai: true,
    admin: true,
    credits: true,
  },

  pricing: {
    currency: "USD",
    plans: [
      {
        id: "free",
        name: "Free",
        price: { monthly: 0, yearly: 0 },
        credits: 100,
        features: ["100 AI credits/month", "Basic support"],
      },
      {
        id: "pro",
        name: "Pro",
        price: { monthly: 19, yearly: 190 },
        credits: 5000,
        features: ["5,000 AI credits/month", "Priority support"],
        popular: true,    // Highlighted on the pricing page
      },
    ],
  },

  theme: {
    defaultMode: "system",
    primaryColor: "#6366f1",
  },

export default config;

Provider System

The core differentiator of OpenBoil is the provider abstraction. Every service (auth, payments, database, AI, email) is defined as a TypeScript interface. Implementations are swapped at runtime based on your config.

How It Works

// 1. Interface defines the contract (types.ts)
export interface AuthProvider {
  signIn(email: string, password: string): Promise<AuthSession>;
  // ... other methods
}

// 2. Factory creates the right implementation (index.ts)
export async function getAuthProvider(): Promise<AuthProvider> {
  switch (config.providers.auth) {
    case "firebase":  return new FirebaseAuthProvider();
    case "supabase":  return new SupabaseAuthProvider();
    default:          return new CustomAuthProvider();
  }
}

// 3. Application code uses the factory (never the implementation directly)
const auth = await getAuthProvider();
const session = await auth.signIn(email, password);
✅
Zero unused code: Provider implementations are dynamically imported. If you use Stripe, the LemonSqueezy code is never loaded. This keeps your bundle clean.

Authentication Providers

custom Custom (JWT + Google OAuth)

Fully functional auth using bcrypt for password hashing and JWT for session tokens. Includes Google OAuth.

  • Passwords: Hashed with bcrypt (12 salt rounds)
  • Sessions: JWT tokens stored in HTTP-only secure cookies (7-day expiry)
  • Google OAuth: Standard OAuth2 code exchange flow
  • Password Reset: Time-limited reset tokens with email delivery
  • Env vars needed: JWT_SECRET, optionally GOOGLE_CLIENT_ID + GOOGLE_CLIENT_SECRET

firebase Firebase Auth

Uses Firebase Admin SDK for server-side auth. Requires npm install firebase-admin.

  • Server-side session verification with verifyIdToken()
  • Google OAuth via Firebase's built-in provider
  • Password sign-in must happen client-side via Firebase Client SDK

supabase Supabase Auth

Uses Supabase GoTrue REST API directly (no SDK dependency).

  • Email/password via /auth/v1/signup and /auth/v1/token
  • Google OAuth via Supabase's built-in OAuth flow
  • Admin operations via service role key

Payment Providers

stripe Stripe

Full Stripe integration using direct REST API calls (no SDK).

  • Checkout: Creates Stripe Checkout Sessions for subscriptions
  • Portal: Customer billing portal for self-service management
  • Webhooks: Handles customer.subscription.created/updated/deleted and invoice.payment_failed
  • Signature verification: HMAC-SHA256 via Web Crypto API

lemonsqueezy LemonSqueezy

LemonSqueezy integration via JSON:API endpoints.

  • Checkout: Creates checkout via variant IDs
  • Webhooks: Handles subscription_created/updated/cancelled/payment_failed
  • Env vars: LEMONSQUEEZY_VARIANT_ID_{PLAN}_{INTERVAL} for each plan

Database Providers

drizzle Drizzle + PostgreSQL Recommended

Full type-safe ORM with complete schema. This is the most production-ready option.

AI Features

Three AI tools are available in the dashboard:

Content Generator (/ai/content)

  • Template selection: Blog Post, Marketing Copy, Product Description, Email, Custom
  • Temperature control (0.0 precise → 1.0 creative)
  • Real-time streaming via Server-Sent Events
  • Cost: 1 credit per generation

Image Generator (/ai/image)

  • Text-to-image generation (OpenAI only — DALL-E 3)
  • Size options: 256x256, 512x512, 1024x1024
  • Generated images displayed in a gallery
  • Cost: 5 credits per image

Document Analyzer (/ai/document)

  • Upload text files (.txt, .md, .csv, .json, .xml, .html)
  • Three modes: Summarize, Extract Key Points, Q&A
  • Cost: 3 credits per analysis

Middleware

The middleware (src/middleware.ts) runs on every request and handles:

  1. Session verification: Reads the session cookie, verifies the JWT token, and populates Astro.locals.user
  2. Route protection: Redirects unauthenticated users to /auth/login for private routes
  3. Admin protection: Routes under /admin/* require role: "admin"

State Management

OpenBoil uses Nano Stores for cross-island state in Astro's island architecture. Stores are in src/stores/.

// Reading state in React components
import { useStore } from "@nanostores/react";
import { $user } from "@/stores/auth";

function MyComponent() {
  const user = useStore($user);
  return <p>Hello, {user?.name}</p>;
}

Deploy to Vercel

Push your code to GitHub

git init && git add . && git commit -m "Initial commit"
git remote add origin https://github.com/you/your-repo.git
git push -u origin main

Import to Vercel

Go to vercel.com/new, import your GitHub repo. Vercel auto-detects Astro via vercel.json.

Set environment variables

In the Vercel dashboard, go to Settings → Environment Variables and add all your .env values. Set DEPLOY_TARGET=vercel.

Deploy

Vercel will build and deploy automatically on every push to main.

Deploy to Netlify

Push your code to GitHub

Same as Vercel — push to a GitHub repo.

Import to Netlify

Go to app.netlify.com/start and import your repo. Netlify reads netlify.toml automatically.

Set environment variables

In Site Settings → Environment Variables, add all your .env values. DEPLOY_TARGET is set to netlify in netlify.toml.

Deploy

Deploys automatically on push.

💡
For Netlify, install the adapter first: npm install @astrojs/netlify. For Vercel: npm install @astrojs/vercel.

Deploy with Docker

Using Docker Compose (recommended)

This starts both the app and PostgreSQL:

# Build and start
docker-compose up --build

# The app runs at http://localhost:4321
# PostgreSQL runs at localhost:5432

Using Dockerfile Only

# Build the image
docker build -t openboil .

# Run the container
docker run -p 4321:4321 --env-file .env openboil
⚠
When using Docker, make sure DATABASE_URL points to the Docker network hostname (e.g., postgresql://openboil:openboil@db:5432/openboil when using docker-compose).
